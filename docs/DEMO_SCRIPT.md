# WARP Prototype Demo Script

## 1. Demo Goal
This demo script is designed to present WARP as a polished, interactive prototype that demonstrates the value of a gamified virtual workspace for remote creative teams.

The demo should focus on clarity, visual progression, and role-based differentiation.

## 2. Demo Flow Overview
Recommended sequence:
1. Show role selection
2. Demonstrate Employee flow
3. Demonstrate Employer flow
4. Demonstrate office progression and level up
5. End with summarized product value

## 3. Demo A - Role Selection

### Objective
Introduce the two user roles and establish role-based differentiation.

### Steps
1. Open WARP landing page
2. Show two role options:
   - Employee
   - Employer
3. Briefly explain that each role has different interaction focus and privileges

### Expected Result
The audience understands that WARP supports multiple user types with different functions.

## 4. Demo B - Employee Flow

### Objective
Show the interactive workspace from the employee perspective.

### Steps
1. Enter as Employee
2. Arrive in the virtual office
3. Move avatar through workspace
4. Hover over a teammate avatar
5. Show teammate activity tooltip or status
6. Click teammate avatar
7. Show interaction menu
8. Send Kudos
9. Trigger or join simulated call domain
10. Show status changing to "In a Call"
11. Open task panel
12. View assigned task
13. Start task
14. Complete task
15. Show reward popup, coins, or points increment

### Expected Result
The audience sees:
- interactive office movement
- lightweight social and team interaction
- gamified task loop
- visible reward response

## 5. Demo C - Employer Flow

### Objective
Show management-side controls and room-building logic.

### Steps
1. Switch to Employer role
2. Enter employer workspace
3. Open Create Room feature
4. Fill interactive form:
   - number of employees
   - number of rooms
   - working hours
   - project timeline
5. Submit form
6. Show build room summary
7. Show dummy invite link or copy action
8. Open task assignment UI
9. Assign task to teammate or user

### Expected Result
The audience sees:
- role difference
- management controls
- room setup simulation
- task assignment authority

## 6. Demo D - Office Level Up

### Objective
Show progression and environmental reward mechanics.

### Steps
1. Show office current state before progression
2. Complete sufficient tasks or trigger progress threshold
3. Trigger office level up
4. Show asset changes:
   - upgraded floor or furniture
   - additional decorations
5. Compare before and after office state

### Expected Result
The audience understands that workspace progression is tied to task completion and team productivity.

## 7. Demo E - Avatar Customization

### Objective
Show personal expression and visual identity features.

### Steps
1. Open avatar customization panel
2. Change:
   - hair
   - skin tone
   - face preset
   - top
   - bottom
3. Show updated avatar preview

### Expected Result
The audience sees that users can personalize their visual identity in the workspace.

## 8. Presenter Notes
During the demo, the presenter should emphasize:
- WARP is a prototype, not a production product
- some features are simulated intentionally
- the core goal is to demonstrate interaction design and system concept
- the value lies in making remote teamwork feel more visible, engaging, and motivating

## 9. Recommended Demo Timing
Suggested total duration:
- Role selection: 1 minute
- Employee flow: 3 to 4 minutes
- Employer flow: 2 to 3 minutes
- Office progression: 1 to 2 minutes
- Avatar customization: 1 minute
- Closing: 1 minute

Total recommended duration:
8 to 12 minutes

## 10. Fallback Plan
If a feature becomes unstable during live demo:
- use the pre-prepared route or state
- focus on the visible concept rather than hidden technical detail
- skip optional transitions and continue with the next core interaction
